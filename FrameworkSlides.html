Symfony Structure (modified)

Organization

  • Naturally organized
  • Separation of your code and libraries
  • Separation of Models, logic, templates (MVC)
  • Can be difficult in CLI editors, but excels in IDEs (PHPStorm)

Folder structures like this also improve security

Symfony Structure
  • app – Configs and Overrides
  • src – Your Code
    • Bundles
      • Controllers – Logic
      • Dependency Injection – Specify Configs, Det. Dependencies
      • Resources – Routing, Services, and Templates
      • Tests
  • vendor – Libraries
    • Symfony Core
    • Monolog
    • Doctrine/Propel
    • Twig
    • Swiftmailer
    • UEcode (example)
  • web
    • Css
    • JS
    • app.php – Initialize the framework
  • bin – scripts
  • composer.json & composer.lock

Libraries

Third party libraries are a huge piece of programming. Frameworks make it easy to organize them.

Good frameworks use Composer!

Say you want to use monolog in your project...

Without Composer

  1. Find the download for monolog
  2. Download it to your library directory
  3. Load all the classes (this is the hard step)
  4. ???
  5. Profit

With Composer

  1. Find package on Packagist
  2. 					        composer require monolog/monolog:1.3.x-dev
  3. Done.

Code Reuse

My Favorite Benefit!

With a framework, you are able to take full use of classes and their extensions. Symfony (along with others) also suggest using services for logic that needs to be reused.

// src/Project/AcmeBundle/Service/CampaignService.php
namespace Project\AcmeBundle\Service;

use \Project\AcmeBundle\Service
use \Project\AcmeBundle\Entity\User;

class CampaignService extends Service
{
	// Trait containing database info (php 5.4 awesomeness)
	use DatabaseAwareTrait;

	public function getCampaigns( User $user )
	{
		$repo = $this->getEntityManager()
		        ->getRepository( 'ProjectAcmeBundle:Campaign' );
		// Shorthand array syntax is also php 5.4 awesomeness
		return $repo->findAllBy( [ 'user' => $user ] );
	}
}

And the Usage!

// src/Project/AcmeBundle/Controller/CampaignController.php
namespace Project\AcmeBundle\Controller;
use \Project\AcmeBundle\Controller;
use \Project\AcmeBundle\Entity\User;

class CampaignController extends Controller
{
	public function indexAction()
	{
		$user = $this->getUser();

		// Defined in configs (couple places possible)
		$cs = $this->get( 'campaign_service' );
		$campaigns = $cs->getCampaigns( $user );
		return $this->render(
		    'PlatformAcmeBundle:Campaign:index.html.php',
		    [ 'campaigns' => $campaigns' ]
		);
	}
}
